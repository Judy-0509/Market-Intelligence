# Humanize Korean — AI 한글 티 제거 (Fast/단일 패스, vendored)

> **벤더링 안내:** 이 스킬은 [github.com/epoko77-ai/im-not-ai](https://github.com/epoko77-ai/im-not-ai)
> (commit `14aeb52d13e737beb4e999cb7cb92275d0969689`, 2026-07-07 fetch)의
> `.claude/skills/humanize-korean/`을 가져온 것이다. 원본은 Fast(단일 호출 monolith
> 에이전트)와 Strict(5인 에이전트 파이프라인) 두 모드를 지원하지만, 이 벤더링본에는
> **Fast 모드만 포함**한다. Strict 모드와 그 하위 에이전트 정의(`agents/*.md`)는
> code/pipeline 자산이라 제외했고, Fast 모드도 원래는 `humanize-monolith`라는 별도
> 에이전트를 호출했으나 그 에이전트 정의 역시 제외 대상이므로, 아래 절차는 **하위
> 에이전트 호출 없이 이 스킬을 실행하는 모델이 직접 한 번에(단일 패스) 수행**하도록
> 고쳤다. 규칙 내용(quick-rules.md의 탐지·처방·자체검증·등급 기준표)은 원본 그대로다.
> 정밀 검증(Strict 5인 파이프라인)이 필요하면 원본 저장소에서 전체 플러그인을 설치한다.
> 수정 내역 전체 목록은 `writing-skills/README.md`의 "humanize-korean 수정 내역"
> 참고.

## 절차 (단일 패스)

### 1단계: run_id 준비

1. cwd 기준 `_workspace/` 폴더 아래 오늘 날짜로 새 작업 폴더를 만든다:
   `_workspace/{YYYY-MM-DD-NNN}/`.
2. 기존 시퀀스 확인은 `Glob` 도구로 한다:
   `Glob(pattern="_workspace/YYYY-MM-DD-*/01_input.txt")` → 결과에서 폴더명의 NNN을
   추출해 최댓값을 찾는다. 오늘 날짜 폴더가 없으면 NNN = 001, 있으면 최댓값 + 1.
   (주의: Glob은 디렉터리 자체를 매칭하지 못하므로 반드시 `01_input.txt`처럼 폴더
   안의 파일을 패턴에 포함한다. `Bash ls`는 사용하지 않는다.)
3. 작업 시작 시 다음 한 줄을 사용자에게 출력한다:
   `humanize-korean (vendored, fast) — run_id: {YYYY-MM-DD-NNN}`

### 2단계: 입력 저장 + 장르 추정

1. 입력 텍스트를 `_workspace/{run_id}/01_input.txt`에 저장한다.
2. 입력 첫 300자로 장르를 자동 추정한다 (칼럼 | 리포트 | 블로그 | 공적). 사용자가
   장르를 명시했으면 그 값을 우선한다.

### 3단계: 탐지 + 윤문 + 자체검증 (하위 에이전트 없이 직접 수행)

1. `references/quick-rules.md`를 전체 읽는다.
2. quick-rules.md의 **Do-NOT 목록**(고유명사·제품명·모델명·기관명, 수치·날짜·단위,
   큰따옴표 안 직접 인용, 법률 조문, 수학·화학·통계 표기, 업계 표준 영어 약어)에
   해당하는 부분은 절대 수정하지 않는다.
3. quick-rules.md A~J 카테고리 표의 패턴을 입력 텍스트에서 탐지한다.
4. 탐지된 각 패턴을 표의 "처방" 칸에 적힌 대로 윤문한다.
5. 윤문 직후 quick-rules.md "자체검증 체크리스트" 6개 항목을 점검한다. 하나라도
   위반이면 해당 부분(edit)만 롤백하고 다시 윤문한다. 이 재시도는 문서 전체에서
   **최대 1회**까지만 허용한다. 그래도 미해결이면 결과를 그대로 두고 summary에
   "자가검증 미통과 항목 N건"을 표기한다.
6. 변경률을 계산한다: (수정된 글자 수 ÷ 원문 글자 수) × 100.
   - 30% 초과 → summary에 경고 문구 추가.
   - 50% 초과 → **윤문을 중단**하고 원문을 그대로 반환 + 중단 사유를 안내한다.
7. quick-rules.md "등급 기준" 표(A/B/C/D)로 자가 채점한다.

### 4단계: 결과 파일 작성

1. `_workspace/{run_id}/final.md`에 윤문본을 쓴다 (3단계에서 50% 초과로 중단했다면
   원문을 그대로 쓴다).
2. `_workspace/{run_id}/summary.md`에 다음을 표로 정리한다: 변경률(%), 등급(A~D),
   탐지된 카테고리 목록(quick-rules.md의 ID), 자체검증 6항목 각각의 통과 여부.

### 5단계: 결과 전달

사용자에게 다음 4가지를 반환한다:
1. 한 줄 상태: `완료. 변경률 X% / 등급 Y / 자체검증 N/6 통과`
2. 윤문본 본문 (마크다운 블록)
3. summary.md의 핵심 표
4. 등급이 B 이하면: "정밀 검증이 필요하면 원본 저장소
   (github.com/epoko77-ai/im-not-ai)에서 Strict 모드(5인 파이프라인)를 별도
   설치하세요." 안내

## 부분 재실행 / 후속 명령

| 사용자 신호 | 처리 |
|---|---|
| "특정 카테고리만 다시" | 새 run_id로 1~5단계 재실행, 해당 카테고리 ID만 대상으로 지정 |
| "이 문단만" | 해당 문단만 입력으로 새 run_id 생성 후 1~5단계 재실행 |
| "2차 윤문" | 기존 run_id의 `final.md`를 새 입력으로 새 run_id 생성 후 재실행 |
| "윤문 강도 조정" | `강도` 옵션 변경 후 새 run_id로 재실행 |
| "장르 바꿔서" | 장르 값 변경 후 새 run_id로 재실행 |

## 옵션 (인자 끝에 자연어로)

- `장르: 칼럼|리포트|블로그|공적` — 장르 명시 (생략 시 자동 추정)
- `강도: 보수|기본|적극` — 윤문 강도. 보수 = S1 패턴만 적용, 기본(기본값) = S1+S2
  전체 적용, 적극 = S1+S2 전체 적용 + F/G 카테고리(과도한 수식·헤징) 우선순위 상향
- `최소심각도: S1|S2` — 탐지 임계값 (기본값: S2)

## 주의 사항

- **의미 불변이 최상위 불문율.** 위반 즉시 롤백.
- **수치·고유명사·직접 인용은 탐지/윤문 대상 아님.** Do-NOT 목록 엄수.
- **장르 이탈 금지.** 칼럼이 에세이로, 에세이가 문학으로 옮겨가지 않는다.
- **register 보존.** 격식체 입력 → 격식체 출력.
- **변경률 30% 초과 → 경고, 50% 초과 → 강제 중단.**
- **자동 로드 금지.** 프로젝트 CLAUDE.md 등 다른 파일을 자동 파싱해 옵션을 추론하지
  않는다.
