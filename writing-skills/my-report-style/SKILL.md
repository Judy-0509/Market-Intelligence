# my-report-style — 학습된 문체로 보고서 작성

## 절차

### 1단계: 프로파일 읽기 + 학습 여부 확인

1. 이 SKILL.md와 같은 폴더에 있는 `style-profile.md`를 읽는다.
2. 헤더의 "학습 문서 수"를 확인한다.
3. **학습 문서 수 = 0이면 (또는 파일 자체가 없으면) 다음 문구를 그대로 출력하고
   즉시 중단한다. 보고서를 작성하지 않는다.**

```
아직 학습된 문체가 없습니다. 먼저 style-extractor 스킬로 과거 보고서 1편 이상을
학습시켜 주세요 (예: "이 보고서로 문체 학습해줘"). 학습 후 다시 요청하시면 그
문체로 작성해 드립니다.
```

4. 학습 문서 수가 1 이상이면 2단계로 진행한다.

### 2단계: 우선순위 정리

프로파일의 24개 필드를 신뢰도 등급에 따라 정렬해 참고한다.
- `확정` 필드 → 반드시 적용.
- `잠정` 필드 → 적용하되, 확정 필드와 충돌하면 확정을 우선한다.
- `상충` 필드 → 프로파일에 적힌 "주 스타일" 값을 기본으로 적용한다. 프로파일에
  적힌 예외 조건(예: "○○ 유형 문서에서")에 이번 보고서가 해당하면 소수 값을 대신
  적용한다. 해당 여부가 애매하면 주 스타일을 그대로 쓴다.
- `(미학습)` 필드 → 적용할 규칙이 없으므로 무시하고 일반적인 사내 보고서 관행을
  따른다.

### 3단계: 작성 — 구조 먼저, 그다음 문장/어휘/기호

다음 순서로 적용한다 (반드시 이 순서: 구조 → 문장·어휘 → 숫자·기호).

1. **구조 (STR-1~STR-6)를 먼저 정한다.**
   - STR-1(개조식/서술식 비율)에 맞춰 전체 형식을 개조식·서술식·혼합 중 하나로 정한다.
   - STR-2(두괄식 여부)에 맞춰 결론·핵심 메시지를 앞/뒤 중 어디에 둘지 정한다.
   - STR-3(번호 체계)에 맞춰 목차 번호 체계를 그대로 사용한다.
   - STR-4(제목 스타일)에 맞춰 제목·소제목 형태를 정한다.
   - STR-5(요약/결론 위치)에 맞춰 요약·결론 단락 위치를 정한다.
   - STR-6(표/그림 참조 방식)에 맞춰 표·그림이 있다면 참조 표현을 맞춘다.
2. **문장·어휘 (SEN-1~SEN-4, VOC-1~VOC-5)를 적용한다.**
   - SEN-1(종결어미 분포)의 비율대로 종결어미를 섞는다.
   - SEN-2(평균 길이), SEN-3(불릿당 정보 수)에 맞춰 문장·불릿 길이와 밀도를 맞춘다.
   - SEN-4(능동/피동 경향)에 맞춰 서술어 태를 정한다.
   - VOC-1(연결어)에 기록된 표현을 전환/인과/추가 자리마다 그대로 사용한다.
   - VOC-2·VOC-4(용어 쌍)에 기록된 쌍은, 그 쌍의 두 용어 중 프로파일에 기록된 쪽만
     사용한다 (`혼용`으로 기록된 쌍은 어느 쪽을 써도 된다). 한 슬롯에 두 값 이상이
     근거 수와 함께 나열되어 있으면(예: `증가 2편 / 상승 1편`) 근거 수가 가장 큰
     쪽을 쓰고, 최대 근거 수가 동률이면 혼용으로 취급해 아무 쪽이나 하나만 골라
     문서 전체에서 일관되게 쓴다. 프로파일에 기록되지 않은 쌍은 제약 없음.
   - VOC-3에 등록된 "사용 안 함" 표현은 절대 쓰지 않는다.
   - VOC-5(예시 표현)에 있는 짧은 관용구는 구 하나당 문서 전체에서 최대 1회, 요약
     (서두) 문단 또는 결론·건의 문단 중 한 곳에만 사용한다. 뜻이 맞는 자리가 없으면
     그 구는 사용하지 않는다.
   - TONE-1(단정/헤징 정도), TONE-2(화법 구분)에 맞춰 문체 톤을 정한다.
3. **숫자·기호 (NUM-1~NUM-4, SYM-1~SYM-3)를 적용한다.**
   - NUM-1(단위 표기), NUM-2(증감 표기), NUM-3(괄호 부연 습관), NUM-4(소수점 자릿수)를
     그대로 따라 숫자를 표기한다.
   - SYM-1(기호와 용도), SYM-2(볼드/밑줄 습관), SYM-3(들여쓰기 단계)를 그대로 따른다.

### 4단계: 자체 점검 (확정 필드만 대상)

작성이 끝나면 프로파일에서 신뢰도가 `확정`인 필드를 모두 나열하고, 각 필드마다
"완성된 보고서가 이 값을 지켰는가?"를 예/아니오로 점검한다. 최소한 다음 항목은
`확정` 여부와 무관하게 항상 점검한다:

```
[ ] 종결어미가 프로파일(SEN-1) 비율과 맞는가?
[ ] 번호 체계가 프로파일(STR-3)과 맞는가?
[ ] 숫자 표기(단위·증감·소수점)가 프로파일(NUM-1~NUM-4)과 맞는가?
[ ] 기호 사용(SYM-1~SYM-3)이 프로파일과 맞는가?
```

하나라도 "아니오"이면 해당 부분만 다시 고쳐 쓰고 같은 항목을 재점검한다. 모두
"예"가 될 때까지 반복한다(최대 2회 재점검, 2회 후에도 안 맞으면 결과를 그대로
전달하되 어떤 항목이 남았는지 사용자에게 한 줄로 알린다).

### 5단계: 결과 전달

완성된 보고서를 전달한다. 마지막에 다음 안내를 1줄 덧붙인다 (humanize-korean이
설치되어 있지 않아도 이 스킬 동작에는 영향 없음 — 선택 사항일 뿐):

```
AI 티가 남아있다고 느껴지면 humanize-korean 스킬(설치되어 있는 경우)로 한 번 더
윤문할 수 있습니다.
```
