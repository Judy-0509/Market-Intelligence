# 프로파일 병합 규칙 (Merge Rules)

`references/checklist.md`로 문서 1편(또는 여러 편)을 채운 결과를 `style-profile.md`에
합치는 방법이다. **필드마다 아래 절차를 순서대로, 예외 없이** 적용한다. 판단이 필요한
부분은 남겨두지 않았다 — 애매하면 "동일 여부 판정" 표를 그대로 따른다.

## 0. 필드 유형표 (동일 여부 판정에 사용)

각 필드는 아래 4가지 유형(A, B1, B2, C) 중 하나다. 유형에 따라 "이번 문서 값 = 기존
프로파일 값"인지 판정하는 방법이 다르다.

| 유형 | 필드 ID | 동일 여부 판정 방법 |
|---|---|---|
| A. 범주형 (문자열 완전일치, 단일 값) | STR-2, STR-3, STR-4, STR-5, STR-6, SYM-2, SYM-3, NUM-3, NUM-4, TONE-2 | 두 값의 대표 분류 문자열(예: "두괄식", "명사형", "3단계", "해당 없음", "있음(용도: 단순 수치 부연)")이 정확히 같으면 동일. 다르면 다름. 이 유형은 필드 하나당 값이 항상 1개뿐이다(목록이 아님) — SYM-2·NUM-3처럼 "있음/없음 + 세부 유형" 형태이거나, STR-6처럼 대표 예시 하나만 남기는 필드가 해당된다. |
| B1. 비율형(%) (퍼센트포인트 비교) | STR-1, SEN-1, SEN-4, TONE-1 | 두 값 모두 "%"로 표기된 필드. 각 구성 비율끼리 **10%p 이내로 차이나면 동일**로 간주. 10%p 넘게 차이 나면 다름. (예: 기존 "능동 70%/피동 30%", 이번 "능동 65%/피동 35%" → 5%p 차이 → 동일. 기존 "능동 70%", 이번 "능동 40%" → 30%p 차이 → 다름.) |
| B2. 수량형(절대값/범위) (상대 비율 비교) | SEN-2, SEN-3 | "자", "개" 등 절대 수치·범위로 표기된 필드. **%p가 아니라 기존 평균값 대비 상대 차이**로 비교한다: `|새 평균 − 기존 평균| ÷ 기존 평균 × 100`이 20% 이내면 동일, 초과하면 다름. (예: 기존 평균 25자, 이번 평균 25자 → 차이 0% → 동일. 기존 평균 25자, 이번 평균 40자 → 차이 60% → 다름.) |
| C. 목록형 (항목별 개별 추적, 여러 값이 함께 쌓임) | NUM-1, NUM-2, SYM-1, VOC-1, VOC-2, VOC-3, VOC-4, VOC-5 | 이번 문서에서 나온 개별 항목이 기존 목록에 이미 있으면 "재확인"(그 항목만 근거 +1), 없으면 "신규 항목"(그 항목만 잠정 1편으로 목록에 추가). 항목 단위로 판단하며, 목록 전체를 하나의 값으로 비교하지 않는다. 이 유형은 필드 하나에 서로 다른 값 여러 개가 동시에 존재할 수 있다(예: 단위 표기가 "억원"과 "만 건" 둘 다 쓰일 수 있음). |

**VOC-1·VOC-2·VOC-4 슬롯 특별 규칙:** 이 세 필드는 고정된 하위 슬롯을 가진 목록형이다
— VOC-1은 "전환/인과/추가" 3개 슬롯, VOC-2는 `checklist.md`의 고정 후보 목록 10쌍이
각각 1개 슬롯, VOC-4는 고정 후보 목록 8쌍이 각각 1개 슬롯. 병합 시 슬롯별로 완전히
독립된 목록처럼 취급한다:
- 다른 슬롯의 값과는 절대 비교하지 않는다 (예: VOC-1 인과 슬롯의 "이에 따라"와 전환
  슬롯의 "다만"은 무관).
- 같은 슬롯 안에서: 이번 값이 기존 값과 같으면 그 슬롯만 근거 +1, 다르면 두 값을 함께
  나열하고 각자 근거 수를 유지한다 (예: VOC-2 "증가/상승" 슬롯에 `증가 2편 / 상승 1편`).
  `혼용`도 하나의 값으로 취급한다.
- 이번 문서에서 기록되지 않은(미출현) 슬롯은 건드리지 않는다.

**유형 판정 요령(새 필드를 추가할 때):** 문서마다 이 필드의 값이 "여러 개가 동시에 성립할
수 있는 목록"인가(예: 단위 표기, 기호 종류 — 그렇다면 C), 아니면 "문서 하나당 대표값이
정확히 하나만 나오는 분류/상태"인가(그렇다면 %면 B1, 절대 수치면 B2, 그 외 문자열이면 A)로
구분한다.

## 0-1. 재학습 방지 선행 점검 (문서마다 체크리스트 작성 전에 반드시 수행)

이 점검은 문서 1편에 대한 어떤 분석보다도 먼저, 즉 그 문서의 체크리스트를 채우기
전에 수행한다 (`style-extractor` SKILL.md 3단계 1번과 같은 시점).

**비교에 쓸 문서 제목은 다음 규칙으로 정한다:**
- 파일로 받았으면: 파일명을 제목으로 쓴다.
- 붙여넣기 텍스트면: 마크다운 제목 줄(`#`으로 시작)이 있으면 그 줄을 제목으로 쓴다
  (줄 맨 앞의 `#` 기호들과 그 뒤 공백은 제목에서 제외한다). 없으면 첫 줄을 후보로 하되, **첫 줄이 다음 중 하나에라도 해당하면 제목으로 쓰지
  않고 사용자에게 제목을 물어 그 답을 그대로 쓴다**: (a) 20자 초과, (b) 서술형
  어미(~다/~함/~임/~됨/~음)로 끝남, (c) 숫자·단위·증감 표기(%, %p, 억, 만, +, △)를
  포함. (내용이 담긴 첫 줄이 학습 로그에 영구히 남는 것을 막기 위한 규칙이다.)
- **제목 비교 전 정규화:** 파일명은 경로와 확장자를 제거하고, 연속 공백은 하나로
  줄인 뒤 비교한다.

그 제목을 다음 두 곳과 비교한다 (정규화한 제목 문자열의 완전일치):
- 프로파일 학습 로그의 "학습 내용" 칸에 이미 기록된 문서 제목들 (따옴표 안 문자열)
- **같은 요청 안에서 이미 처리한(앞 순서 문서의) 문서 제목들**

1. **같은 제목이 어디에도 없으면** → 정상 학습. 체크리스트를 채우고 아래 "1. 필드별
   처리 절차"로 진행한다.
2. **같은 제목이 이미 있고, 사용자가 이번 요청에서 "강제 재학습"이라고 명시하지
   않았으면** → 이 문서는 학습하지 않는다. 체크리스트를 채우지 않고 건너뛰며,
   사용자에게 다음 문구를 그대로 출력한다:

```
이미 학습된 문서 제목입니다. 다시 학습하려면 '강제 재학습'이라고 말해 주세요.
```

3. **같은 제목이 이미 있고, 사용자가 "강제 재학습"을 명시했으면** → 학습을 진행하되
   다음 3가지 규칙을 적용한다 (재학습 모드):
   - 어떤 값도 빼거나 감산하지 않는다.
   - "1. 필드별 처리 절차"에서 **"동일하면" 경로(1-2 2번, 1-3 2번, 1-4 2번의 근거
     +1)는 전부 건너뛴다** — 값이 기존과 같은 필드·항목은 근거 문서 수를 올리지 않고
     그대로 둔다 (같은 문서를 두 번 세지 않기 위함). 값이 실제로 새로운 경우(1-1 신규
     기입, 1-4 3번 신규 항목, "다르면" 상충 처리)만 정상 반영한다.
   - "4. 버전·학습 로그 갱신"에서 **"학습 문서 수"를 올리지 않고**, 그 문서의 로그
     줄 학습 내용 칸 끝에 `(재학습)`을 붙인다. 예:
     `v8 | 2026-07-07 | 문서 "6월 실적보고" 학습 (재학습) | 변경: 없음(값 변화 없음)`
     — 재학습에서 아무 값도 바뀌지 않았다면 변경 칸에 `없음(값 변화 없음)`이라고
     적는다 (재학습 로그 줄에만 허용되는 예외).

## 1. 필드별 처리 절차

`checklist.md`로 얻은 24개 필드 값을 STR-1부터 TONE-2까지 순서대로 하나씩 처리한다.
각 필드가 유형 A/B1/B2/C 중 무엇인지는 위 "0. 필드 유형표"에서 확인한다.

### 1-0. "판단 불가"는 병합하지 않는다
이번 문서의 체크리스트 값이 `판단 불가`(또는 `판단 불가`로 시작하는 문자열, 예: `판단
불가(예시 없음, 15자 제한)`)이면 이 필드는 건드리지 않고 그대로 다음 필드로 넘어간다.
프로파일 값도, 근거 문서 수도, 신뢰도 등급도 바꾸지 않는다 (기존에 `(미학습)`이었으면
`(미학습)`으로 그대로 둔다 — "판단 불가"를 마치 발견된 패턴인 것처럼 근거로 쌓지 않는다).

### 1-1. 프로파일 값이 `(미학습)`인 경우
→ 이번 문서의 값을 그대로 적어넣는다.
→ 신뢰도 = `잠정(1편)`, 근거 문서 수 = `1`.
→ 다음 필드로 이동.

### 1-2. 프로파일에 이미 값이 있는 경우 (유형 A: 범주형)
1. 위 유형표의 방법으로 동일 여부를 판정한다.
2. **동일하면**: 근거 문서 수 +1. 근거 문서 수가 3 이상이면 신뢰도를 `확정`으로 올린다
   (3 미만이면 `잠정(N편)`으로 유지, N은 새 근거 문서 수).
3. **다르면 (상충)**: 아래 "2. 상충 처리"로 이동.

### 1-3. 프로파일에 이미 값이 있는 경우 (유형 B1: 비율형 % / B2: 수량형)
1. 필드가 B1인지 B2인지 위 유형표에서 확인하고, 해당하는 기준(B1=10%p, B2=상대
   차이 20%)으로 동일 여부를 판정한다.
2. **동일하면**: 근거 문서 수 +1. 값은 두 관찰치의 가중 평균으로 갱신한다 — 계산식은
   B1/B2 공통으로 `새 값 = (기존 값×기존 근거 문서 수 + 이번 값)÷(기존 근거 문서 수+1)`,
   반올림해 정수로. (예: B1 기존 "70%" 근거 2편, 이번 "65%" → (70×2+65)÷3 반올림 =
   68%. B2 기존 "평균 25자" 근거 1편, 이번 "25자" → (25×1+25)÷2 = 25자.) 범위형 값
   (예: SEN-2의 최소~최대)은 두 문서의 최솟값 중 더 작은 값, 최댓값 중 더 큰 값으로
   갱신한다. 근거 문서 수 3 이상이면 신뢰도 `확정`.
3. **다르면 (상충)**: 아래 "2. 상충 처리"로 이동.

### 1-4. 프로파일에 이미 값이 있는 경우 (유형 C: 목록형)
1. 이번 문서에서 나온 항목마다 기존 목록에 있는지 확인한다.
2. **이미 있는 항목**: 그 항목의 근거 문서 수만 +1 (3편 이상이면 그 항목만 `확정`
   표시).
3. **새 항목**: 목록 끝에 `<항목>(잠정 1편)`으로 추가. 기존 항목은 건드리지 않는다.
4. 목록형 필드는 항목이 서로 달라도 "상충"으로 취급하지 않는다 (예: 연결어 "반면"과
   "다만"이 둘 다 있어도 정상 — 둘 다 유지).
   - 예외: VOC-3(회피 표현)에서 `checklist.md`의 교차 확인 결과 기존에 `확정`으로
     등록된 회피 표현이 이번 문서에 실제로 등장했다면, 이는 상충으로 취급한다 →
     "2. 상충 처리"로 이동.

## 2. 상충 처리 (모든 유형 공통)

1. 기존 값을 **절대로 지우거나 덮어쓰지 않는다.**
2. 두 값을 다음 형식으로 합쳐 적는다:
   `<기존 값 표현> <기존 근거>편 / <새 값 표현> <새 근거>편 → 주 스타일 <근거 문서 수가
   더 많은 쪽>, <근거 문서 수가 적은 쪽>은 <이번 문서 제목 또는 특징>에서`
   - 예: `개조식 4편 / 서술식 1편 → 주 스타일 개조식, 서술식은 "이사회 보고" 유형
     문서에서`
3. 근거 문서 수는 각 값별로 따로 유지한다 (합산하지 않는다).
4. 신뢰도 등급 = `상충`.
5. **기존 값이 이미 `확정`이었던 경우**: 확정 값을 `주 스타일`로 유지하고, 새 값은
   소수 사례로만 각주 형태로 추가한다. 확정 등급 자체를 상충으로 강등하지 않는다.
   단, 확정이 아니었던 값(`잠정`)과 충돌한 경우에는 위 2번 형식대로 `상충` 등급으로
   전환한다.

## 3. 삭제 규칙

- 필드 삭제(값을 다시 `(미학습)`으로 되돌리는 것)는 **사용자가 채팅에서 특정 패턴이
  틀렸다고 명시적으로 말한 경우에만** 수행한다.
  - 예: 사용자가 "그건 아니고 사실 서술식으로 써" 라고 말하면 STR-1을 `(미학습)`으로
    되돌리고 학습 로그에 `삭제: STR-1 사용자 정정`을 남긴다.
- 그 외의 경우(새 문서 값이 다르다는 이유만으로는) 절대 필드를 삭제하지 않는다. 반드시
  "2. 상충 처리"로 남긴다.

## 4. 버전·학습 로그 갱신 (모든 문서 처리 후 마지막에 1회)

1. 프로파일 상단의 "프로파일 버전"을 1 올린다 (예: v6 → v7). 여러 문서를 한 번에
   학습해도 버전은 요청당 1만 올린다.
2. "학습 문서 수"를 이번에 처리한 문서 편수만큼 더한다. **단, 재학습 모드(0-1의 3번)로
   처리된 문서는 편수에서 제외한다.**
3. 학습 로그 표 맨 아래에 **이번에 학습한 문서 1편마다 한 줄씩** 정확히 아래 형식으로
   추가한다 (여러 편을 한 번에 학습했으면 문서 수만큼 줄을 추가하고, 모든 줄에 같은
   {새 버전}을 쓴다. 재학습 모드 문서는 0-1의 3번에 정의된 `(재학습)` 표기를 붙인다):

```
v{새 버전} | {오늘 날짜, YYYY-MM-DD} | 문서 "{문서 제목}" 학습 | 변경: {그 문서로 인해 변경된 필드ID와 한글 요약, 쉼표로 구분}
```

   - 예: `v7 | 2026-07-07 | 문서 "6월 실적보고" 학습 | 변경: 종결어미 확정, 기호 ☞ 추가`
   - "변경" 칸에는 **그 문서로 인해** 값이 바뀌었거나(신규 학습·상충·확정 승격) 근거
     문서 수가 늘어난 필드만 적는다. 최소 1개 이상 반드시 포함한다 (문서를 학습했다면
     최소 "근거 문서 수 증가"라도 발생하기 때문. 예외: 재학습 줄은 0-1의 3번에 따라
     `없음(값 변화 없음)` 허용).
4. 문서 제목·날짜만 학습 로그에 남긴다. **원문 내용·문장은 학습 로그에 절대 옮기지
   않는다.**

## 5. 사용자에게 보여줄 요약 (병합 후 매번 출력)

병합이 끝나면 다음 형식으로 diff 스타일 요약을 사용자에게 보여준다 (프로파일 파일
자체가 아니라 채팅 응답으로):

```
[문체 학습 완료] 프로파일 v{이전 버전} → v{새 버전}

신규 학습: <필드ID 목록>
확정 승격: <필드ID 목록> (없으면 "없음")
상충 발견: <필드ID 목록> (없으면 "없음")
학습 문서 수: {이전 수} → {새 수}
```
