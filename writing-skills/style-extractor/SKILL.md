# style-extractor — 문체 학습 스킬

이 스킬은 사용자의 과거 보고서를 분석해 `../my-report-style/style-profile.md`에 문체
규칙을 누적한다. **"학습"은 모델 재훈련이 아니라 이 프로파일 파일에 규칙을 쌓는
것이다.** 아래 절차를 번호 순서대로, 빠짐없이 실행한다. 판단이 필요한 지점은 없다 —
모든 판정 기준은 `references/checklist.md`와 `references/merge-rules.md`에 이미
정의되어 있다.

## 절차

### 1단계: 원문 입력받기 + 보안 고지

사용자에게 분석할 보고서 텍스트를 요청한다. 텍스트를 직접 붙여넣거나 `.md`/`.txt`
파일 경로를 주도록 안내한다. 여러 편을 한 번에 줄 수도 있다.

작업을 시작하기 전에 반드시 아래 문구를 사용자에게 그대로 출력한다:

```
원문은 어떤 파일에도 저장하지 않고, 분석 후 즉시 버립니다. 프로파일에는 문체 규칙과
15자 이내 짧은 예시 구만 남고, 문장 전체나 문서 내용은 절대 저장되지 않습니다.
```

파일 경로를 받았다면 `Read` 도구로 해당 파일 내용을 읽어온다. 분석이 끝나면(4단계
완료 후) 원문 텍스트를 대화 컨텍스트 밖 어디에도 저장하지 않는다 — 새 파일로
쓰지 않고, 원문을 복사한 임시 파일을 만들지 않는다.

### 2단계: 프로파일 위치 확인

1. 이 스킬 폴더 기준 상대 경로 `../my-report-style/style-profile.md`를 확인한다
   (두 스킬은 같은 skills 루트 아래 나란히 설치되어 있어야 한다. 예:
   `~/.claude/skills/style-extractor/`와 `~/.claude/skills/my-report-style/`).
2. 파일이 있으면 그대로 읽어 이후 단계에서 사용한다.
3. 파일이 없으면, 이 SKILL.md 맨 아래 "부록: 빈 프로파일 템플릿"의 내용을 그대로
   복사해 `../my-report-style/style-profile.md` 경로에 새로 만든다. (이 스킬은
   `my-report-style` 폴더가 없어도 부록만으로 자기 완결적으로 동작한다 — 단, 이
   경우에도 파일은 반드시 `../my-report-style/style-profile.md` 경로에 만든다.
   폴더가 없으면 먼저 만든다.)

### 3단계: 문서별 체크리스트 작성

받은 문서마다 (여러 편이면 편별로 반복):

1. **재학습 점검 (다른 무엇보다 먼저, 체크리스트를 채우기 전에):** 문서 제목을
   정한다 — 제목을 정하는 규칙(파일명/마크다운 제목 줄/사용자 질문 분기)과 비교 전
   정규화는 `references/merge-rules.md` "0-1. 재학습 방지 선행 점검"에 정의되어
   있으니 그 규칙을 그대로 따른다 (여기에 요약을 두지 않는다 — 0-1이 유일한 정본).
   그 제목이 프로파일 학습 로그 또는 **같은 요청에서 이미 처리한 앞 문서들** 중에
   있는지 0-1의 기준대로 확인한다. 이미 있는 제목인데 사용자가 이번 요청에서
   "강제 재학습"을 명시하지 않았으면, 이 문서는 체크리스트를 채우지 않고 건너뛰며
   0-1에 정의된 안내 문구를 그대로 출력한 뒤 다음 문서로 넘어간다.
2. `references/checklist.md`를 연다.
3. 24개 필드(STR-1~STR-6, SEN-1~SEN-4, NUM-1~NUM-4, SYM-1~SYM-3, VOC-1~VOC-5,
   TONE-1~TONE-2)를 순서대로 **전부** 채운다. 근거가 없는 필드는 `판단 불가`라고
   적는다 — 건너뛰지 않는다.
4. 각 필드는 체크리스트에 정의된 "출력 형식"을 그대로 따른다.
5. 1번에서 정한 문서 제목과 오늘 날짜를 따로 기록해둔다 (학습 로그용).
   강제 재학습으로 진행 중인 문서라면 "재학습 모드"라는 사실도 함께 기록해둔다
   (4단계 병합과 학습 로그 표기에 필요).

### 4단계: 프로파일에 병합

1. `references/merge-rules.md`를 연다.
2. 3단계에서 채운 24개 필드 값을 병합 규칙 절차(필드 유형 판정 → 동일/상충 처리)에
   따라 하나씩 `style-profile.md`에 반영한다. 재학습 모드 문서는 merge-rules.md
   0-1의 3번 규칙(동일 값 근거 미가산, 학습 문서 수 미증가, `(재학습)` 로그 표기)을
   적용한다.
3. 여러 문서를 한 번에 받았다면 문서를 하나씩 순서대로 병합한다 (동시에 섞어서
   처리하지 않는다).
4. 병합이 끝나면 `merge-rules.md` 4번(버전·학습 로그 갱신)에 따라 프로파일 버전을
   올리고 학습 로그에 **문서 1편마다 한 줄씩** 추가한다. 로그에는 문서 제목·날짜·
   변경된 필드ID만 남긴다 — **문서 내용은 로그에 절대 옮기지 않는다.**
5. **병합된 문서가 0편이면**(전부 재학습 거부로 건너뛴 경우 등) 이 4단계의 버전
   갱신·학습 로그 추가를 전부 건너뛰고, 학습된 문서가 없다는 사실만 사용자에게
   알린다 (이 경우 5단계 요약도 생략한다).

### 5단계: 변경 요약 보고

`merge-rules.md` 5번(사용자에게 보여줄 요약) 형식 그대로 사용자에게 diff 스타일
요약을 출력한다. 프로파일 파일 자체를 화면에 전부 출력할 필요는 없다 — 요약 형식만
출력하면 된다.
