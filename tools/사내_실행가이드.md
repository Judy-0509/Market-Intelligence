# 사내 엑셀 구조 프로파일링 실행 가이드

TAM 지원 시스템 Phase 1(DB 자동화)을 설계하려면 사내 엑셀들의 구조를 알아야 합니다.
이 폴더의 스크립트들은 엑셀을 **읽기 전용**으로 열어 구조 정보만 추출합니다. 원본 파일은
수정하지 않습니다 (읽기 전용 + 매크로 실행 차단 + 외부 링크 갱신 차단).

## 0. 어떤 스크립트를 쓸까

결과물을 사내망 **밖으로 반출할 때 글자수 제한이 있는 채널**(채팅 등)을 쓰는 경우가 많아서,
기본값은 용량이 작은 텍스트 브리프입니다.

| 모드 | 스크립트 | 출력 | 언제 |
|---|---|---|---|
| **경량 모드 (기본)** | `Get-ExcelBrief.ps1` | 압축 텍스트 (수십~수백자 수준) | 평소 기본 흐름, 반출 글자수 제한이 있을 때 |
| 상세 모드 (선택) | `Get-ExcelProfile.ps1` | JSON (훨씬 큼) | 텍스트 브리프로 부족해서 더 상세한 구조가 필요할 때 |

## 1. 준비 (공통)

필요한 `.ps1` 파일을 사내 PC로 복사 (git clone 또는 텍스트 복사-붙여넣기).
스크립트 내용은 수정하지 말 것 (인코딩 문제 방지를 위해 일부러 영문으로 작성됨).

## 2. 경량 모드 (기본) — `Get-ExcelBrief.ps1`

- 별도 설치 불필요 — Excel이 설치된 Windows PC면 바로 동작
- 기본 흐름: **① 시트 목록 확인 → ② 분석할 시트 선택 → ③ 그 시트만 텍스트로 추출**
- 추출 내용: 시트 목록/크기(전체), 선택 시트의 헤더행·샘플 행 2개·마지막 행·열 타입·수식 개수/예시·표/피벗,
  워크북 레벨의 이름정의·외부 연결(파워쿼리·링크 포함) 개수

### 방법 A: 사내 Claude Code(Sonnet)에 아래 프롬프트 붙여넣기

```
엑셀 구조를 텍스트 브리프로 뽑아줘. 반드시 아래 순서대로만 진행해.

1) 먼저 아래 명령으로 시트 목록을 확인해:
   powershell -ExecutionPolicy Bypass -File "<스크립트 경로>\Get-ExcelBrief.ps1" -Path "<엑셀 파일 경로>" -ListOnly

2) 출력된 시트 목록(번호/이름/숨김여부)을 나에게 그대로 보여주고,
   어떤 시트를 분석할지 나에게 물어봐. 절대 네가 임의로 시트를 고르지 마.

3) 내가 시트를 지정하면 그 시트만 추출해:
   powershell -ExecutionPolicy Bypass -File "<스크립트 경로>\Get-ExcelBrief.ps1" -Path "<엑셀 파일 경로>" -Sheets "<내가 지정한 시트명>"
   만약 시트명이 안 먹으면(한글/특수문자) 시트 번호로:
   ... -SheetIndex <시트 번호>

4) 실행이 끝나면 생성된 텍스트 브리프의 내용을 채팅에 그대로 출력해줘
   (용량이 작아서 파일 없이 채팅으로 바로 복사해 나가면 됨).
   반출 채널에 글자수 제한이 있으면 -MaxChars <제한치>를 붙여서 여러 조각으로
   나눠 달라고 요청해 (조각마다 [p i/N] 표시가 붙음).
   스크립트를 수정하거나 결과를 요약하지 마.
```

`<스크립트 경로>`와 `<엑셀 파일 경로>`만 실제 경로로 바꾸면 됩니다.

> **대상 선정 팁**: ① 취합용 `tam.xlsx` ② 조사기관 원본 엑셀(Omdia/IDC 등, 기관별 최소 1개씩)
> 파일마다 위 과정을 반복하고, `-OutFile brief_tam.txt` / `-OutFile brief_omdia.txt`
> 처럼 결과 파일명을 나눠 저장하세요.

### 방법 B: PowerShell에서 직접 실행

```powershell
# 1단계: 시트 목록만 확인
powershell -ExecutionPolicy Bypass -File .\Get-ExcelBrief.ps1 -Path "D:\TAM\tam.xlsx" -ListOnly

# 2단계: 원하는 시트만 텍스트로 추출 (시트명 또는 번호)
powershell -ExecutionPolicy Bypass -File .\Get-ExcelBrief.ps1 -Path "D:\TAM\tam.xlsx" -Sheets "전망시트"
powershell -ExecutionPolicy Bypass -File .\Get-ExcelBrief.ps1 -Path "D:\TAM\tam.xlsx" -SheetIndex 2

# 반출 채널 글자수 제한이 있으면 조각으로 분할
powershell -ExecutionPolicy Bypass -File .\Get-ExcelBrief.ps1 -Path "D:\TAM\tam.xlsx" -Sheets "전망시트" -MaxChars 1500
```

### 주요 옵션 (`Get-ExcelBrief.ps1`)

| 옵션 | 기본값 | 설명 |
|---|---|---|
| `-Path` | (필수) | 엑셀 파일 또는 폴더 경로 |
| `-ListOnly` | 꺼짐 | **시트 목록만** 출력 (번호/이름/숨김). 시트 선택용 사전 단계 |
| `-Sheets` | (전체) | 지정한 시트명만 상세 추출. 예: `-Sheets "Forecast"` |
| `-SheetIndex` | (전체) | 시트를 번호로 지정 (1부터, `-ListOnly` 출력의 `[번호]`). 시트명 인코딩 문제 시 대안 |
| `-DataRows` | 2 | 헤더 다음에 보여줄 샘플 행 수 |
| `-MaxColsShown` | 25 | 행 미리보기에 보여줄 최대 열 수 |
| `-CellChars` | 18 | 셀 값 하나를 자를 최대 길이 (초과 시 `..`로 표시) |
| `-MaxChars` | 0 | 0보다 크면 결과를 이 글자수 이하 조각(part) 파일로 분할. 반출 채널 글자수 제한 대응용 |
| `-OutFile` | excel_brief.txt | 결과 파일명 |
| `-Recurse` | 꺼짐 | 폴더 지정 시 하위 폴더까지 스캔 |
| `-MaskNumbers` | 꺼짐 | **기밀 수치 보호**: 숫자를 `#N`으로 마스킹 (1900~2100 정수는 연도로 간주해 유지) |

시트를 지정해도 **시트 목록(`==` 줄)은 항상 전체 시트를 보여주므로**, 어떤 시트가 더 있는지는
결과만 봐도 알 수 있습니다.

### 전달 전 확인 (보안 체크리스트, 경량 모드)

1. 생성된 `excel_brief.txt`(또는 조각 파일들)를 메모장으로 열어 훑어보기
   - 텍스트 셀 값(고객사명, 프로젝트 코드명 등)이 그대로 들어갑니다
   - 수치가 민감하면 `-MaskNumbers`를 켜고 다시 실행
2. 문제없으면 텍스트 내용을 채팅에 그대로 붙여넣기 (용량이 작아 파일 첨부 없이도 전달 가능)
   - 한글이 실제 한글 글자로 보이는 것이 정상입니다 (JSON과 달리 이스케이프되지 않음)
   - 반출 채널에 글자수 제한이 있으면 `-MaxChars`로 나눈 조각(`_p1`, `_p2`, ...)을 순서대로 전달
   - ⚠️ 이 텍스트를 GitHub 저장소(public)에 커밋하지 마세요 — 채팅으로만 전달

## 3. 상세 모드 (선택) — `Get-ExcelProfile.ps1`

텍스트 브리프로는 부족하고 더 상세한 구조(전체 열 서식, 더 많은 샘플 행/수식 등)가 필요할 때만
사용합니다. 결과가 JSON이라 경량 모드보다 훨씬 큽니다 (반출 글자수 제한이 있으면 부적합).

### 방법 A: 사내 Claude Code(Sonnet)에 아래 프롬프트 붙여넣기

```
엑셀 구조 프로파일링을 도와줘. 반드시 아래 순서대로만 진행해.

1) 먼저 아래 명령으로 시트 목록을 확인해:
   powershell -ExecutionPolicy Bypass -File "<스크립트 경로>\Get-ExcelProfile.ps1" -Path "<엑셀 파일 경로>" -ListOnly

2) 출력된 시트 목록(번호/이름/크기)을 나에게 그대로 보여주고,
   어떤 시트를 분석할지 나에게 물어봐. 절대 네가 임의로 시트를 고르지 마.

3) 내가 시트를 지정하면 그 시트만 프로파일링해:
   powershell -ExecutionPolicy Bypass -File "<스크립트 경로>\Get-ExcelProfile.ps1" -Path "<엑셀 파일 경로>" -Sheets "<내가 지정한 시트명>"
   만약 시트명이 안 먹으면(한글/특수문자) 시트 번호로:
   ... -SheetIndex <시트 번호>

4) 실행이 끝나면 콘솔 출력을 그대로 보여주고,
   생성된 excel_profile.json 의 전체 경로를 알려줘.
   스크립트를 수정하거나 결과를 요약하지 마.
```

`<스크립트 경로>`와 `<엑셀 파일 경로>`만 실제 경로로 바꾸면 됩니다.

> **대상 선정 팁**: ① 취합용 `tam.xlsx` ② 조사기관 원본 엑셀(Omdia/IDC 등, 기관별 최소 1개씩)
> 파일마다 위 과정을 반복하고, `-OutFile profile_tam.json` / `-OutFile profile_omdia.json`
> 처럼 결과 파일명을 나눠 저장하세요.

### 방법 B: PowerShell에서 직접 실행

```powershell
# 1단계: 시트 목록만 확인
powershell -ExecutionPolicy Bypass -File .\Get-ExcelProfile.ps1 -Path "D:\TAM\tam.xlsx" -ListOnly

powershell -ExecutionPolicy Bypass -File .\Get-ExcelProfile.ps1 -Path "D:\TAM\tam.xlsx" -Sheets "전망시트"
powershell -ExecutionPolicy Bypass -File .\Get-ExcelProfile.ps1 -Path "D:\TAM\tam.xlsx" -SheetIndex 2
```

### 주요 옵션 (`Get-ExcelProfile.ps1`)

| 옵션 | 기본값 | 설명 |
|---|---|---|
| `-Path` | (필수) | 엑셀 파일 또는 폴더 경로 |
| `-ListOnly` | 꺼짐 | **시트 목록만** 출력 (번호/이름/크기/숨김). 시트 선택용 사전 단계 |
| `-Sheets` | (전체) | 지정한 시트명만 상세 프로파일링. 예: `-Sheets "Forecast"` (여러 개: 쉼표 구분) |
| `-SheetIndex` | (전체) | 시트를 번호로 지정 (1부터, `-ListOnly` 출력의 `[번호]`). 시트명 인코딩 문제 시 대안 |
| `-Recurse` | 꺼짐 | 폴더 지정 시 하위 폴더까지 스캔 |
| `-OutFile` | excel_profile.json | 결과 파일명 |
| `-SampleRows` | 8 | 시트 상단 샘플 행 수 |
| `-MaxCols` | 40 | 샘플 최대 열 수 |
| `-MaskNumbers` | 꺼짐 | **기밀 수치 보호**: 숫자를 `#NUM`으로 마스킹 (1900~2100 정수는 연도로 간주해 유지) |

시트를 지정해도 **시트 목록(sheetList)은 항상 JSON에 포함**되므로, 어떤 시트가 더 있는지는
결과만 봐도 알 수 있습니다.

### 전달 전 확인 (보안 체크리스트, 상세 모드)

   - 수치가 민감하면 `-MaskNumbers`를 켜고 다시 실행

## 알려진 제약

- 암호 걸린 파일은 열지 못하고 결과에 `error`로 기록됨 (암호 알려주면 옵션 추가 가능)
- `.xlsb`, `.xls` 구형 포맷도 지원 (Excel COM 사용이라 Excel이 여는 건 전부 가능)
